## 背景

Qt 客户端已经可以登录后读取运行监控列表，但一线维护人员还不能从客户端写入设备运行状态。为了让客户端覆盖“查看状态 -> 记录状态 -> 刷新确认”的闭环，需要接入后端现有 `POST /api/v1/monitoring/states` 接口。

## 变更内容

- 在 Qt API 客户端中新增运行状态写入方法，提交设备、状态、严重度和指标摘要。
- 在运行监控页增加状态写入表单，并在写入成功后刷新列表。
- 更新本地启动、HTTP 服务和生产级说明文档，明确客户端已接入运行状态写入。

## 影响范围

- 客户端运行监控页从只读列表扩展为可写入状态。
- 后端接口不新增路由，复用已有运行监控写入能力。
- 离线演示模式保持可读兜底；写操作要求后端登录态。
