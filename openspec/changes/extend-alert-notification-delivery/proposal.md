## 背景

告警规则已经可以在告警创建时生成通知记录，但通知仍停留在 `queued` 状态，缺少投递尝试、失败原因、投递时间和重试入口。生产级工业支持系统需要把通知作为可审计 outbox 处理，先完成状态流转和接口边界，再逐步接入真实邮件、Webhook、短信或企业协同平台。

## 变更内容

- 告警通知记录新增尝试次数、失败原因和投递时间字段。
- 后端新增通知队列投递和单条通知重试服务方法。
- 后端新增 `POST /api/v1/alert-notifications/dispatch` 和 `POST /api/v1/alert-notifications/{id}/retry`。
- MySQL 新增 006 迁移登记并扩展 `alert_notifications` 表。
- Qt 告警通知对话框新增投递队列、重试选中和投递状态字段展示。
- HTTP 冒烟测试覆盖通知投递状态流转。

## 影响范围

- 当前投递器是本地 outbox 适配边界：支持 `console`、`email`、`webhook` 通道的状态流转，不直接连接外部网络服务。
- 后续真实外部投递器可以沿用现有状态字段、重试入口和审计响应继续扩展。
