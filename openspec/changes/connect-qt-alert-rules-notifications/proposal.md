## 背景

告警中心已经具备创建、查询和生命周期处置能力，但规则配置与通知记录仍停留在规划层。工业支持系统需要在告警产生时依据设备、级别和启用状态自动生成可追溯通知记录，便于值班、审计和后续接入邮件、Webhook、短信等投递通道。

## 变更内容

- 后端新增告警规则领域模型、仓储能力和 HTTP 接口。
- 后端新增告警通知记录模型，创建告警时按规则自动生成通知记录。
- MySQL 新增 `alert_rules` 与 `alert_notifications` 表和 005 迁移登记。
- Qt 告警中心新增“规则”和“通知”入口，支持规则新增与通知查看。
- 集成测试覆盖规则创建、规则查询、告警命中通知和参数/权限校验。

## 影响范围

- 当前通知先持久化为 `queued` 状态，不直接执行外部投递，为后续真实投递器、重试队列和审计留出边界。
- 告警创建接口保持原响应结构，新增副作用为生成匹配规则的通知记录。
