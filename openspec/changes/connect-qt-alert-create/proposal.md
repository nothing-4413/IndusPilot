## 背景

Qt 客户端已经可以查看和处置告警，也可以从告警生成维护工单，但还不能由操作员手动登记新的现场告警。后端已经提供 `POST /api/v1/alerts`，客户端需要补齐告警创建入口，让演示流程覆盖“发现异常 -> 登记告警 -> 处置/生成工单”的完整链路。

## 变更内容

- 在 Qt API 客户端中封装告警创建接口。
- 在告警中心增加“创建告警”操作，填写告警编号、设备编号、级别、状态、标题和负责人。
- 创建成功后刷新告警列表，并展示后端操作结果。
- 更新 README、本地启动、HTTP 服务和生产级说明，标记 Qt 告警创建已接入。

## 影响范围

- 客户端告警中心从只读处置扩展为可登记告警。
- 后端接口不新增路由，复用现有 `POST /api/v1/alerts`。
- 写操作要求后端登录态和 `alert:write` 权限；离线模式仍只提供演示数据。
