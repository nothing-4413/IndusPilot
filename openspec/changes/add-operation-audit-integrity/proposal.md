# Change: 操作审计哈希链完整性校验

## Why

操作审计已经支持查询、分页和 CSV 导出，但生产环境还需要证明记录没有被静默篡改。哈希链可以把每条审计事件与上一条事件绑定，帮助管理员在复盘、合规检查和异常排查时快速发现断链位置。

## What Changes

- 操作审计事件新增 `previousHash` 与 `eventHash` 字段，记录时自动生成 SHA-256 哈希链。
- MySQL 审计表新增哈希字段与索引，并登记 `009_operation_audit_integrity_schema` 迁移。
- 后端新增 `GET /api/v1/audit/integrity`，复算审计链并返回校验状态、记录总数、断点事件和最新哈希。
- 审计 JSON 与 CSV 导出包含哈希字段。
- Qt“操作审计”页面新增完整性状态、校验按钮和哈希列。
- 基础测试与 HTTP smoke 测试覆盖哈希生成、权限拒绝和完整性校验。

## Non-Goals

- 不实现外部时间戳服务、区块链存证或第三方公证。
- 不自动修复历史缺失哈希的旧审计记录。
- 不改变审计保留周期、归档策略或导出审批流程。
