# Change: 生产就绪缺口清理

## Why

当前后端 API 链路已经完整，但仍存在配置项未驱动运行时行为、开发账号硬编码、部分文档占位和前端静态演示数据等问题。为了让项目更接近生产级展示，需要按优先级收敛这些缺口。

## What Changes

- 增加配置驱动的仓储选择，默认保留内存模式，允许切换到 MySQL 仓储。
- 将身份认证接入 `UserRepository` 和 `PermissionRepository`，减少硬编码开发账号对运行时的影响。
- 明确仍未接入真实存储或真实 AI 的模块边界，并修正文档占位/乱码。
- 让配置文件、文档和运行时行为保持一致。

## Non-Goals

- 不在本次实现完整密码哈希策略。
- 不在本次实现所有模块的 MySQL/MongoDB/Redis 仓储。
- 不调用真实大模型服务。
