## 背景

告警处置和维护工单已经分别接入 Qt 客户端，但操作员仍需要手动复制告警编号再创建工单。后端已经提供 `POST /api/v1/work-orders/from-alert`，可以根据告警生成关联工单，因此客户端需要补齐告警到工单的联动入口。

## 变更内容

- 在 Qt API 客户端中封装从告警生成工单接口。
- 在告警中心增加“生成工单”操作，基于选中告警填写处置摘要。
- 成功生成后刷新维护工单列表，并展示后端操作结果。
- 更新文档，将“工单一键从告警生成”从待接入能力调整为已接入能力。

## 影响范围

- 客户端告警中心新增跨模块操作入口。
- 后端接口不新增路由，复用现有 `POST /api/v1/work-orders/from-alert`。
- 写操作要求后端登录态和 `work-order:write` 权限；离线模式保留演示数据但不伪造写入成功。
