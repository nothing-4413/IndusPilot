# productionize-backend-platform

## Why

Foundation 已经完成领域边界、Qt 客户端骨架、基础数据脚本、Redis 会话存储和 AI 辅助边界。下一阶段需要把后端从“模块骨架”推进到可部署、可测试、可运维的平台服务，支撑后续工业协议、工单闭环和 AI 实用模块持续接入。

## What Changes

- 引入真实 HTTP 服务边界，替换当前进程内 Router 演示接口。
- 完善配置加载，支持环境变量覆盖、密钥外置、依赖连接参数和启动校验。
- 建立 MySQL 仓储接口，逐步替换内存集合和开发种子数据。
- 将 Redis session store 接入应用启动配置，并提供依赖健康检查。
- 增加认证中间件和权限守卫，保护资产、告警、工单和 AI 接口。
- 增加结构化日志、错误码、请求追踪和基础指标出口。

## Impact

- 后端将从库式业务模块变为可启动服务。
- 测试会从单进程单元测试扩展到接口级集成测试。
- 配置、部署和本地启动文档需要同步更新。
