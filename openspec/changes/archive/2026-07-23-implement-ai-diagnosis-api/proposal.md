# Change: 实现 AI 辅助诊断 HTTP API

## Why

资产、运行监控、告警和维护工单已经形成基础业务链路。为了体现项目的 AI 增强能力，需要提供可审计、非阻塞的 AI 辅助诊断入口，即使外部 AI 服务未启用，也要返回明确不可用状态并记录交互。

## What Changes

- 扩展 AI 服务，支持诊断请求输入校验和交互记录查询。
- 新增 AI 故障排查、日志摘要和交互记录 HTTP API。
- 保持 AI 建议非权威、非阻塞，并记录 relatedType/relatedId/prompt/output。
- 扩展集成测试和开发文档。

## Non-Goals

- 不直接调用真实大模型服务。
- 不自动执行任何维修动作或控制指令。
- 不把 AI 输出作为告警或工单状态变更依据。
