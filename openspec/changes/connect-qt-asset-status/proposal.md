## 背景

Qt 客户端已经可以登录后查看资产列表，但资产生命周期状态仍只能通过后端接口或测试脚本更新。后端已经提供 `PATCH /api/v1/assets/{id}/status`，客户端需要补齐资产状态维护入口，让设备资产模块覆盖“查看资产 -> 调整状态 -> 刷新确认”的基础闭环。

## 变更内容

- 在 Qt API 客户端中封装资产状态更新接口。
- 在资产页面增加“更新状态”操作，基于选中资产选择生命周期状态。
- 状态更新成功后刷新资产列表并展示后端操作结果。
- 更新 README、本地启动、HTTP 服务和生产级说明，标记 Qt 资产状态更新已接入。

## 影响范围

- 客户端资产页面从只读列表扩展为可维护生命周期状态。
- 后端接口不新增路由，复用现有 `PATCH /api/v1/assets/{id}/status`。
- 写操作要求后端登录态和 `asset:write` 权限；离线模式仍只提供演示数据。
