# Change: 操作审计筛选分页

## Why

操作审计事件已经完成端到端展示，但记录量增长后，仅返回最近数组会降低排查效率。管理员需要按操作者、动作、资源类型和结果快速定位关键操作，并通过分页降低客户端和 HTTP 响应压力。

## What Changes

- 后端操作审计服务新增查询条件，支持按 `actor`、`action`、`resourceType` 和 `result` 精确筛选。
- `GET /api/v1/audit/events` 兼容原数组响应；传入 `limit` 或 `offset` 时返回分页对象 `{ items, total, limit, offset }`。
- Qt “操作审计”页面新增用户、动作、资源类型、结果筛选和上一页/下一页分页控件。
- HTTP 集成测试覆盖筛选分页、分页总数和无效分页参数。

## Non-Goals

- 不实现模糊搜索、时间范围查询或服务端排序配置。
- 不改变审计事件写入动作范围。
- 不实现审计 CSV 导出或外部日志归档。
