# Change: 操作审计闭环联机化

## Why

工业支持系统需要追踪关键操作的执行人、动作、对象、结果和链路追踪号。当前登录、告警通知派发等关键动作缺少统一的操作审计视图，Qt 客户端也无法让管理员直接复核这些记录。

## What Changes

- 后端新增操作审计事件模型、服务和仓储接口，支持内存与 MySQL 存储。
- HTTP 新增 `GET /api/v1/audit/events`，通过 `audit:read` 权限保护审计查询。
- 登录成功、告警通知派发和告警通知重试写入操作审计事件。
- Qt 客户端新增“操作审计”页面，登录后同步审计事件，后端不可用时保留离线演示数据。
- MySQL 新增 `007_operation_audit_events_schema.sql` 并纳入部署前预检。

## Non-Goals

- 不实现审计记录删除、编辑或人工补录。
- 不实现按用户、时间、动作的服务端筛选分页。
- 不接入外部 SIEM、日志平台或消息队列归档。
