# Change: Qt 客户端监控与告警联机列表

## Why

Qt 客户端已经完成 HTTP 登录和资产列表同步，但运行监控与告警中心仍停留在静态离线数据。为了让桌面端逐步形成真实运维闭环，需要继续接入只读监控状态和告警列表，同时保持后端不可用时的离线兜底。

## What Changes

- 登录成功后通过 Bearer token 调用 `GET /api/v1/monitoring/states`。
- 登录成功后通过 Bearer token 调用 `GET /api/v1/alerts`。
- 运行监控和告警页面显示后端同步/离线演示模式。
- 请求失败、响应格式异常或会话失效时保留离线数据，并提示当前模式。

## Non-Goals

- 不在本次实现监控状态写入、告警确认/分派/关闭等写操作。
- 不接入维护工单和 AI 诊断页面。
- 不引入异步刷新、自动轮询或 WebSocket。
