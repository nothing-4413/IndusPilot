## 背景

维护工单已经支持创建、分派、开始、完成和关闭，但现场处理过程中仍缺少编辑摘要/处理人/结果的入口，也缺少附件登记能力。生产级运维系统需要把现场照片、检测报告、维修记录等证据作为可追溯元数据挂接到工单。

## 变更内容

- 后端新增工单编辑接口 `PATCH /api/v1/work-orders/{id}`。
- 后端新增工单附件元数据登记和查询接口。
- MySQL 新增 `work_order_attachments` 表和 004 迁移登记。
- Qt 维护工单页面新增“编辑”和“附件”操作。
- 集成测试覆盖工单编辑、附件登记、附件查询和参数校验。

## 影响范围

- 工单编辑只更新摘要、处理人和处理结果，不改变既有创建/流转接口。
- 附件接口保存 URI 和元数据，不直接接收二进制文件，为后续对象存储或 FastDFS 留出边界。
